# Productivity Paradox Slidev Draft

Rough English Slidev deck for an internal 10-15 minute sharing session on Paul A. David's 1990 essay, *The Dynamo and the Computer*, and a cautious extension to AI.

## Files

- `slides.md` - main Slidev deck with speaker notes.
- `assets/electrification-diffusion.svg` - rough diffusion chart from David 1989 Table 3.
- `assets/ai-retrofit-vs-redesign.svg` - rough AI analogy diagram.
- `assets/kammgarnspinnerei-pfaffendorf-1925.jpg` - group-drive / line-shaft reference photo.
- `assets/marconi-individual-driving-motors.jpg` - unit-drive / individual-motor reference photo.
- `assets/ibm-704-edp.jpg` - EDP/mainframe-era reference photo.
- `data/electrification-diffusion.csv` - source data for the chart.
- `notes/source-notes.md` - citations and caveats.
- `notes/speaker-script.md` - fuller spoken script draft.

## Current Draft Principle

Keep it simple: no fancy animation, no dense econ math. The audience should leave with one transferable mechanism:

> Productivity payoff often comes less from bolting a new tool onto an old system, and more from redesigning the system around the new tool.
